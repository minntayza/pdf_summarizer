# a — Summary

- **1. How to Use This Roadmap**: Follow a foundation-first order: Python → LLM concepts → frameworks → advanced agent architecture → production deployment. Practice by reading questions, rewriting answers, implementing projects, and noting failures.

- **2. Python Fundamentals**: Python is default due to ecosystem (FastAPI, Pydantic, agent frameworks). Key features: type hints, dataclasses/Pydantic, async/await, context managers. Project structure: layers (app, core, agents, tools, rag, eval, infra). Test deterministic layers; for LLM steps use golden prompts and metrics. Avoid common pitfalls: unbounded retries, missing timeouts, global state, mixing prompt with business logic.

- **3. LLM Fundamentals**: LLMs predict next token via transformer layers. Tokens affect cost, latency, context. Context window limits require memory strategies (summaries, retrieval, compression). Prompting is interface design: role, task, constraints, output schema. Temperature/top-p control randomness; function calling makes agents reliable. Prompt injection prevention: treat retrieved text as untrusted, separate tool outputs. Reduce hallucination with tools, RAG, constrained outputs, verification loops.

- **4. Framework Selection**: Choose LangGraph for deterministic workflows/graphs, CrewAI for role-based multi-agent, AutoGen for flexible chat. LangGraph is production-ready due to explicit state machines, checkpointing, human-in-the-loop. Anti-pattern: copying demo code; architecture comes first. Avoid vendor lock-in by abstracting providers. "State" in agent graphs is structured, typed data for reproducibility and safety.

- **5. Advanced Framework Concepts**: LCEL composes components into pipelines. Runnables are functional building blocks with uniform interfaces. Workflows include branching, loops, human approvals. Prevent infinite loops with max steps, time budgets, stop conditions. Structured output (JSON schema) is critical for safe tool execution. Use verifier steps with deterministic checks then optional LLM judge. Parallelization tradeoffs: reduce latency vs cost; cap concurrency.

- **6. Memory Management**: Short-term = immediate context; long-term = external stores. Use summaries for session events, embeddings for semantic retrieval, structured memory for stable facts. Checkpointing saves state for resume after failures. Context budgeting allocates window percentages. Evaluate memory with A/B tests. Manage multi-agent memory: shared vs private, with provenance. Failure modes: irrelevant retrieval, noisy storage, feedback loops. Handle user corrections by updating and deprecating old entries.

- **7. Tool Integration**: Agent-friendly tools have clear name, narrow purpose, typed input, deterministic output, fast failure. Safety: least privilege, separate read/write, require confirmations, allowlists, sandboxes. Tool outputs should be concise; use pagination or top-k. Handle errors with structured codes, retryable flags. Version tools like APIs. Prevent unnecessary calls with explicit policies and classifier steps.

- **8. RAG Systems**: RAG reduces hallucinations by injecting external knowledge. Chunk size: 300–800 tokens with 10–20% overlap; use semantic chunking. Dense (embedding) vs sparse (BM25) vs hybrid retrieval. Re-ranking improves precision. Prevent irrelevant retrieval: top-k, metadata filtering, abstention when no evidence. Evaluation metrics: recall@k, precision@k, MRR, factuality, citation correctness. Keep indexes fresh with incremental updates. Metadata filtering is critical for security. Grounded generation requires citations.

- **9. Agents & Multi-Agents**: ReAct pattern interleaves reasoning, actions, observations. Supervisor coordinates specialist agents. Safe communication via structured messages, role boundaries, provenance. Separate planning (low temperature) from execution (schema-validated). Handle uncertainty with confidence scores, ask for clarification. Prevent tool abuse with centralized policy gate and least-privilege tool sets. Agent protocols standardize I/O. Practical multi-agent design: retriever, summarizer, writer, critic with supervisor. Evaluation: task success rate, error buckets, regression tests.

- **10. Real-World Projects**: Architecture: UI → FastAPI → Agent Orchestrator → Tools + RAG + Memory. FastAPI good for async, typing, OpenAPI. Streamlit UI: chat layout, streaming, citations. Docker: slim base, pinned deps, health checks. AWS deployment: ECS/Fargate, RDS/DynamoDB, S3, Secrets Manager, CloudWatch. Observability: structured logs, tracing spans, metrics, replay mechanism. Streaming responses: stream tokens, keep tool calls non-streaming. Security: auth, rate limiting, input validation, prompt injection defenses. CI/CD: lint, test, eval suites.

- **11. Quick Checklist**: Python → LLM fundamentals → framework → advanced concepts → memory → tools → RAG → agents → real projects. Bring 2-3 concrete projects with failures debugged.
