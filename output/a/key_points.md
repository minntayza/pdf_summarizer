# a — Key Exam Points

- 🔴 **High Priority (Must-Know for Exams)**
  - **ReAct Pattern**: Interleave reasoning, action, observation; improves factuality.
  - **Function/Tool Calling**: Structured output vs free text; enables reliable tool execution.
  - **Prompt Injection Prevention**: Treat retrieved text as untrusted; separate system instructions.
  - **Context Budgeting**: Allocate window percentages to instructions, memory, retrieved docs.
  - **Checkpointing**: Save state for resume after failures; critical for long-running agents.
  - **RAG Evaluation Metrics**: recall@k, precision@k, factuality, citation correctness.
  - **LangGraph Advantages**: Explicit state machines, checkpointing, human-in-the-loop.

- 🟡 **Medium Priority (Important to Understand)**
  - **LCEL & Runnables**: Composition of components; modularity and observability.
  - **Multi-Agent vs Single Agent**: Decompose only when beneficial; specialization vs coordination overhead.
  - **Structured Output**: JSON schema validation prevents brittle parsing and hallucination.
  - **Memory Types**: Short-term vs long-term; summaries vs embeddings vs structured.
  - **Hybrid Retrieval**: Combine dense and sparse for better recall.
  - **Supervisor Agent Role**: Coordinates specialists; adds overhead but enables safety gates.
  - **FastAPI for Backend**: Async, Pydantic, OpenAPI; strong typing for agent APIs.

- ⚠️ **Common Pitfalls & Mistakes**
  - **Unbounded retries/loops**: Always set max steps and time budgets.
  - **Mixing prompt logic with business logic**: Leads to risky changes; keep separate.
  - **Missing observability**: Cannot debug "agent went weird" without structured logs and traces.
  - **Copying framework demo code without architecture**: Frameworks amplify chaos if fundamentals are skipped.
  - **Storing outdated or unverified facts in memory**: Causes hallucinations; only store what can be justified.
  - **Not using metadata filtering in RAG**: Leaks data across tenants; must be enforced in code.
  - **High temperature for action flows**: Prefer lower randomness for reliability in tool calls.
  - **Treating user content as trusted**: Always sanitize and enforce system policies to prevent injection.
