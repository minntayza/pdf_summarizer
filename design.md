# Smart PDF Lecture Summarizer — Technical Design

> Phase 1 (MVP) — Single-Agent Architecture
> Last updated: 2026-06-15

---

## 1. System Overview

A local CLI tool that reads academic PDF lecture files, extracts text, sends it to Claude API with a structured prompt, and writes the three-output response (summary, key points, flashcards) as Markdown files.

```
┌──────────┐    ┌──────────┐    ┌───────────┐    ┌───────────┐
│ input/   │───▶│ pdf_     │───▶│ Claude    │───▶│ output/   │
│ lecture  │    │ reader   │    │ API call  │    │ summary   │
│ .pdf     │    │ .py      │    │ (prompts  │    │ key_points│
│          │    │          │    │  .py)     │    │ flashcards│
└──────────┘    └──────────┘    └───────────┘    └───────────┘
                      │                                │
                      ▼                                ▼
                 Text extraction               Markdown files
                 (PyMuPDF)                     written to disk
                                              (utf-8 encoded)
```

## 2. Components

### 2.1 `main.py` — Entry Point

```
python main.py input/lecture.pdf [-o output_dir] [--provider claude|gemini] [--model MODEL]
```

Arguments:
| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `input_path` | positional | required | Path to PDF file |
| `-o, --output` | optional | `output/` | Output directory |
| `--provider` | optional | `claude` | AI provider (`claude` or `gemini`) |
| `--model` | optional | provider default | Model override (e.g. `claude-sonnet-4-6` or `gemini-1.5-flash`) |

Returns exit code 0 on success, non-zero on error.

### 2.2 `pdf_reader.py` — PDF Text Extraction

Responsibilities:
- Accept PDF file path
- Extract all text content page-by-page using PyMuPDF (fitz)
- Return concatenated string with page markers (`[Page N]`)
- Handle: encrypted PDFs, empty pages, non-text pages
- Reject: non-PDF files, corrupted files

Edge cases:
| Condition | Behavior |
|-----------|----------|
| PDF has no extractable text | Raise `NoTextExtractedError` with clear message |
| PDF > 50 pages | Warn user, use chunking strategy (section-by-section) |
| Encrypted PDF | Raise `EncryptedPDFError` — suggest user decrypt first |
| File doesn't exist | Raise `FileNotFoundError` |
| Not a PDF (mime mismatch) | Raise `InvalidFileTypeError` |

### 2.3 `prompts.py` — Prompt Templates

Contains structured system prompt and user prompt templates.

**System prompt** — instructs Claude to act as an academic study aid expert:
- Role: Expert academic tutor
- Output format: Strict JSON with three fields
- Quality guidelines: concise but comprehensive, exam-focused

**User prompt** — structured request embedding the extracted PDF text:
- Instructs Claude to produce a JSON response with exactly three keys:
  1. `summary` — bullet-point structure with key definitions
  2. `key_points` — exam-relevant predictions, common pitfalls, high-yield concepts
  3. `flashcards` — array of Q&A pairs (question/answer format)

Also exports `OUTPUT_SCHEMA` — a JSON Schema that defines the expected response shape for validation.

### 2.4 Chunking Strategy (for PDFs > 50 pages)

When the extracted text exceeds a threshold (configurable, default ~60K tokens):
1. Split text by page ranges (e.g., pages 1-20, 21-40, 41+)
2. Process each chunk sequentially
3. Merge results:
   - `summary`: concatenate with section headers
   - `key_points`: deduplicate by concept, keep top N
   - `flashcards`: deduplicate by question text

**Why sequential not parallel:** Simplifies implementation; PDFs over 50 pages are rare in this project's target use case. If speed becomes an issue, parallelize in Phase 2.

## 3. Output Format Specifications

### `summary.md`
```markdown
# [Lecture Title] — Summary

## [Section/Topic Name]
- Key definition or concept
- Important formula or principle
- Supporting detail

---
> Generated by Smart PDF Lecture Summarizer on 2026-06-15
```

### `key_points.md`
```markdown
# [Lecture Title] — Key Exam Points

## 🔴 High Priority — Likely Exam Questions
### [Topic]
- Point 1
- Point 2

## 🟡 Medium Priority — Important to Know
...same structure...

## ⚠️ Common Pitfalls & Mistakes
- Trap 1
- Trap 2

---
> Generated by Smart PDF Lecture Summarizer on 2026-06-15
```

### `flashcards.md`
```markdown
# [Lecture Title] — Flashcards

## Flashcard 1
**Q:** Question text
**A:** Answer text

## Flashcard 2
**Q:** ...
**A:** ...

---
> Generated by Smart PDF Lecture Summarizer on 2026-06-15
```

## 4. Output Validation

Using Pydantic to validate Claude's JSON response:

```python
class Flashcard(BaseModel):
    question: str
    answer: str

class LectureOutput(BaseModel):
    summary: str
    key_points: str
    flashcards: list[Flashcard]
```

On validation failure:
1. Retry once with a sharper prompt ("Ensure exact JSON format")
2. If still failing, log the raw response and raise `ValidationError`
3. User can re-run with `--debug` to inspect raw API output

## 5. Error Handling Strategy

| Error | User Message | Action |
|-------|-------------|--------|
| File not found | "❌ Could not find file at path/to/file.pdf" | Exit code 1 |
| Not a PDF | "❌ The file must be a PDF" | Exit code 1 |
| No text extracted | "❌ No extractable text found in PDF. This tool works best with text-heavy lecture PDFs." | Exit code 2 |
| PDF too large | "⚠️ PDF has {N} pages — processing in chunks ({K} batches). This may take a few minutes." | Continue with chunks |
| API error | "❌ Claude API error: {message}. Check your API key and network connection." | Exit code 3 |
| Validation error | "❌ Unexpected response format from AI. Try reducing PDF size or retrying." | Exit code 4 |
| All OK | "✅ Done! Files saved to output/{lecture_name}/" | Exit code 0 |

All errors output to stderr, success messages to stdout (Unix convention).

## 6. Configuration

Configuration lives in the code constants at the top of each module, not in a config file (keeping it simple for MVP):

```python
# config.py (Phase 1 — simple constants)
CHUNK_SIZE = 60000  # characters per chunk
MAX_RETRIES = 1
OUTPUT_DIR = "output"
DEFAULT_PROVIDER = "claude"  # "claude" or "gemini"
CLAUDE_MODEL = "claude-sonnet-4-6"
GEMINI_MODEL = "gemini-1.5-flash"
```
