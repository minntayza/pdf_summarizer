# Sketch 002: Library List

## Design Question
How should the document library display your PDFs? Users browse, search, and manage documents here. The design should balance information density with visual appeal.

## How to View
open .planning/sketches/002-library-list/index.html

## Variants
- **A: Rich Cards** — Grid of cards with thumbnail, status badge, subject tag, and provider icon. More visual, better for scanning by appearance. Feels like Notion gallery view.
- **B: Table Rows** — Spreadsheet-like table with columns for document, subject, provider, status, date. Dense information, easy to sort mentally. Feels like Linear's list view.
- **C: Compact List** — Minimal single-line rows with icon, title, and inline metadata. Actions appear on hover. Feels like Raycast's file picker — fast and keyboard-friendly.

## What to Look For
- Information density vs visual appeal balance
- How processing/error states feel
- Subject tags and provider badges visibility
- Action buttons (rename, delete, retry) discoverability
- Mobile responsiveness potential
